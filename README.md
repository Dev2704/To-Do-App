# 📝 To-Do App

A modern To-Do application built with **HTML, CSS, and Vanilla JavaScript**.

This project was created to strengthen my understanding of JavaScript fundamentals, DOM manipulation, state management, and browser storage before moving on to React.

---
## 🌐 Live Demo

🔗 https://dev2704.github.io/To-Do-App/

## ✨ Features

* ➕ Add new tasks
* ❌ Delete tasks
* ✅ Mark tasks as completed
* ⌨️ Add tasks using the Enter key
* ✔️ Input validation
* 💾 Persistent storage using Local Storage
* 🔄 Automatic rendering after every state update
* 🆔 Unique task IDs
* 📱 Clean and responsive dark-themed UI

## 🚀 What I Learned

This project helped me understand how a frontend application manages data and keeps the UI in sync with application state.

Some of the key concepts I practiced include:

* Structuring JavaScript into reusable functions
* Separating rendering logic from application logic
* Managing application state with arrays of objects
* Persisting data using Local Storage
* Debugging real-world issues instead of only following tutorials

## 🤖 About AI

AI was used throughout this project as a **learning assistant**, not as a replacement for programming.

I used it to:

* Understand JavaScript concepts
* Debug issues
* Review code structure
* Discuss different implementation approaches
* Learn best practices and software design

The project's architecture, debugging process, feature implementation, and final decisions were developed through an iterative learning process where I wrote, tested, modified, and improved the code myself.

I believe AI is most valuable when it helps developers learn faster, think more clearly, and write better software—not when it replaces understanding.
